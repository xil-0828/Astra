# 画面設計（URL / ページ構成）｜Astra

## 目的
- アプリ内の全画面とURL構造を明確にする
- Next.js（App Router）の app/ 構成にそのまま対応させる
- 将来の機能追加（チャット）を壊さずに拡張できるようにする

---

## 画面一覧（MVP）

| 画面名 | URL | 内容 |
|---|---|---|
| トップ | / | アニメ検索・一覧表示 |
| アニメ詳細 | /anime/[id] | 作品情報・レビュー一覧・レビュー投稿 |
| ログイン | /login | Googleログイン |
| マイページ | /me | 自分が投稿したレビュー一覧 |

---

## 画面ごとの役割

### トップ（/）
- アニメ作品一覧を表示
- 作品名検索
- 各作品の詳細ページへの導線

### アニメ詳細（/anime/[id]）
- 作品情報の表示
- レビュー一覧の表示
- ログイン済みユーザーのみレビュー投稿可能

### ログイン（/login）
- Googleログインのみ提供
- ログイン後は直前のページへリダイレクト

### マイページ（/me）
- 自分のレビュー一覧を表示
- レビューの編集・削除は将来検討（MVP外）

---

## Next.js app ディレクトリ構成（MVP + 将来拡張込み）

app/
├ page.tsx // トップ
├ login/
│ └ page.tsx // ログイン
├ me/
│ └ page.tsx // マイページ
└ anime/
└ [id]/
├ page.tsx // アニメ詳細（レビュー含む）
└ chat/
└ page.tsx // アニメごとのチャット（将来実装・MVP外）

※ MVPでは chat/page.tsx は実装しない
※ URL構造とディレクトリ構成のみ先に設計しておく

---

## 設計方針
- 1 URL = 1 画面を厳守する
- app/ ディレクトリ構成と URL は常に一致させる
- 将来の機能追加は「階層追加」で対応し、既存ページは壊さない
- MVP段階では不要なページ・機能は作らない
